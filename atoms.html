Usage Guidelines

  • Use primary colors for all interactive elements like buttons and links
  • Accent colors should be used sparingly for secondary actions
  • Semantic colors (success, warning, error) should only be used for their intended purposes
  • Neutral colors are for text, backgrounds, and non-interactive elements
  • Always ensure sufficient color contrast for accessibility

Usage Guidelines

  • Use Display for marketing headlines and hero sections
  • Heading 1-6 should be used in descending order for content hierarchy
  • Subheadings work well for section introductions below main headings
  • Paragraph styles should be used for body content with appropriate weights
  • Caption text is ideal for image captions, table headers, and small labels
  • Always use the mobile sizes for screens smaller than 768px

  • Design should be mobile-first, starting with the smallest breakpoint

  • Test designs on actual devices to ensure proper rendering
  • Consider landscape orientations for tablets and mobile devices

Drop Shadows

X:0, Y:4, Blur:16, Spread:4, Opacity:15%, Color: #272A2B

box-shadow: 0 4px 16px 4px rgba(39, 42, 43, 0.15);

Usage Guidelines

  • Use shadows sparingly to create depth and hierarchy
  • This shadow style should be used for elevated elements like modals and cards
  • Avoid using multiple shadows on a single page to prevent visual noise
  • Shadows should be subtle and not distract from the content
  • Test shadows in different lighting conditions to ensure readability

Tints

Tints are somewhat transparent layers that are used to highlight something on the page, e.g. a modal.

Neutral dark

Use this only behind models and menus.

Neutral regular

Use this to highlight neutral clickable and draggable elements, such as pressed state toggle switch (off-mode).

Usage Guidelines

  • Neutral dark tint should only be used behind modals and menus to focus attention
  • Neutral regular tint works well for interactive element states
  • Ensure tint opacity is sufficient to show the overlay effect but not obscure content completely
  • Tints should be used consistently across the application
  • Consider accessibility when using tints over text or important content
